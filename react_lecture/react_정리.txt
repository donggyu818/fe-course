1. 리액트(React) 의 탄생
    (1) 탄생 배경
    - React 는 페이스북의 개발팀이 화면 업데이트를 위해 개발하던 중 탄생
    - React 는 자바스크립트 기반의 라이브러리 (언어 X)
    - 2013년도에 등장한 선언형 방식으로 UI를 표현함
    - 현재는 2024 버전의 API를 지원 (React.19v) : 항상 최신 버전으로 개발을 권장함!

    (2) 왜 새로운 기술이 필요한가?
    - 기존의 기술 한계점 : 정적인 웹 구축에 초점, 대규모 앱 구축에 부적합.
    - 페이스북의 요청 : 
        다양한 인터렉션(사용자와의 양방향 소통)
        변화가 잦은 대규모 앱이 필요

❗ React 핵심 목적 - 변화가 자주 일어나는 대규모 애플리케이션에 적합한 쉽고 효율적인 라이브러리를 만들자. (실시간 갱신)

2. 리액트의 특징 3가지
    (1) 컴포넌트 기반 (컴포넌트 : 박스들. 예시로 <header>, <div> 등으로 묶여진 것들)
        - UI를 독립적이고 재사용 가능한 컴포넌트 단위로 분리

    (2) 가상돔(Vitual DOM)
        - 실제 DOM(DOM 객체) 을 복제하여 가상돔에서 필요한 부분만 업데이트를 진행한 후 한 번에 실제 DOM에 반영하여 빠른 업데이트를 진행
            예_좋아요 누르면 페이지 전체가 다시 불러와지는 게 아니라 좋아요 부분만 가상돔으로 만들고 업데이트를 진행 후 반영.

    (3) 단방향 데이터 흐름
        - 부모 컴포넌트 -> 자식 컴포넌트로 데이터를 전달함
        - 자식 컴포넌트 -> 부모 컴포넌트로 데이터 전달 불가

3. 리액트 앱(애플리케이션) 생성 = 리액트 프로젝트 생성
    (1) Vite(비트) - 리액트 라이브러리를 제공하는 오픈 API

    (2) 기존 CRA(Create React App) 를 대체하는 라이브러리
        CRA 는 2025년 기준 deprecate

    (3) Vite 를 활용한 리액트 앱을 생성
        1) 개발 환경 구성 - Node.js 가 설치되어 있어야 함. (자바스크립트를 이용하기 때문)
            Ex_ cmder 에 node -v, npm -v 쳐보고 버전 나오는지 확인
        2) 리액트 앱 생성 : my-app
            형식 : npm create vite@latest [앱 이름(프로젝트명)]
            Ex_ npm create vite@latest my-app

5. 컴포넌트
    (1) 정의
        - UI를 구성하는 독립적이고 재사용 가능한 단위
        - 함수가 HTML을 반환하는 구조를 가짐 : return()
        - 레고 블록을 조합하여 페이지를 구성

    (2) 종류
        - 함수형 컴포넌트 : 2018년 이후 사용됨, 현재 표준. 일반 함수, 화살표 함수 사용 가능
            function [컴포넌트명] () {
                return ();
            }
        - 클래스형 컴포넌트 : 거의 사용 안 함
            class [클래스명] extends React.Componet {
                render ();
            }

    (3) 특징
        - 재사용성. 여러 번 사용할 거는 컴포넌트로 만들면 됨.
        - 독립성 : 각각의 컴포넌트를 독립적으로 생성하여 유지보수 향상
        - 조합가능 : 컴포넌트를 조합하여 화면을 구성

    (4) 구성요소
        형식 : 
            function [컴포넌트명] ( Props ) {
            // 로직, 컴포넌트 상태 관리(React Hooks) - 비동기이지만 await 안 붙여도 됨.

                return (
                    <h1>{name}</h1> - js 에서는 ${} 로 붙였지만 여기서는 $ 안 붙여도 됨.
                )
            }
            const [컴포넌트명] = ( Props ) => {
                // 로직, 컴포넌트 상태 관리(React Hooks)

                return (

                )
            }
        - 컴포넌트명은 PascalCase(첫 글자 대문자)
        - 컴포넌트의 인자값은 Props 로 전송되며, 단방향(부모 -> 자식)으로만 진행됨.
        - 로직, 상태관리 등
        - return 함수를 통해 화면 렌더링(출력)
        - return 함수는 JSX 문법이 적용됨

    (5) 부모 컴포넌트 vs 자식 컴포넌트
        - 부모 컴포넌트 : index.html 에 실제 렌더링되는 컴포넌트. 딱 하나만 존재.
        - 자식 컴포넌트 : 부모 컴포넌트의 return 함수에서 호출되는 컴포넌트

    (6) Export(내보내기) / Import(불러오기)
        - 컴포넌트 앞에 export 를 추가함
        형식 :
            export (default) function [컴포넌트명] () {
                return ()
            }
        - 별칭을 이용하여 자식 컴포넌트를 import 함
        - export default 컴포넌트 컴포넌트명. 그대로 사용
        - export 컴포넌트 {컴포넌트명} 형식으로 import
            import [별칭] from '컴포넌트 위치.확장자';

6. JSX (JavaScript Extension) 
    (1) JSX 란?
        - 자바스크립트 + HTML 태그를 포함한 리액트 확장 문법
        - 공식 JS 문법은 아니지만 리액트 개발팀에서 적극 권장함
        - 중괄호 {} 를 사용하여 HTML 안에 자바스크립트 표현식을 사용 가능

    (2) JSX 에서 사용 가능한 표현식
        - 산술 표현식 : {numA + numB}
        - 문자열 표현식 : {strA + strB}
        - 논리 표현식 : {booleanA || booleanB}. 실제 렌더링은 되지 않고 조건식에서 처리 됨.
        - 객체 표현식 : Object Literal 은 객체명.프로퍼티명으로 접근해야 함. 객체명으로는 접근 불가(에러 발생)

    (3) JSX 문법
        - 모든 태그는 반드시 cosing(/>) 태그를 가져야 함.
        - 최상위 태그를 반드시 하나만 가지고 있어야 함. (빈 태그로라도 감싸있어야 함.) 아니면 에러.
        - 최상위 태그를 빈 태그로 <></> 축약해서 사용 가능함.

    (4) 조건부 렌더링 - 삼항연산자, if조건문 사용 가능
        - 삼항 연산자를 활용하여 조건에 맞게 렌더링
        - JSX 에서는 실행한 결과만 표현식

    (5) JSX 스타일링
        - 인라인 스타일링 (태그 안에 바로 들어가는 style) : CSS 속성을 객체로 작성, 카멜케이스로 사용
            예) <h1 style={{backgroundColor:'red', ..}}>Hello~ React</h1>
        - CSS 파일로 작성하고, 컴포넌트에서 import
            예) App.css --> App.jsx import
                App.jsx
                -----------
                import 'CSS 파일 위치';
                    ~
                <h1 className="body"> CSS 클래스명 적용 </h1>

7. 컴포넌트에 값 전달하기
    (1) Props 란?
        - 부모 컴포넌트가 자식 컴포넌트에게 전달하는 단일 *객체*(Properties 약자)
        - 단방향으로만 전달 가능함 : 부모 -> 자식

    (2) Props 전달 방식
        1) 기본적인 전달
            function Parent() {
                return(
                    <Child name="홍길동" />
                )
            }
            function Child(props) { // props= {name: "홍길동"} 형태로 넘어옴. props 는 다른 거로 해도 됨.
                return(
                    <h1>Hello~ My name is {props.name}</h1>
                )
            }
        2) 구조 분해 할당 ( {} => 구조분해 할당 )
            function Parent() {
                return(
                    <Child name="홍길동" />
                )
            }
            function Child({name}){
                return(
                    <h1>Hello~ My name is {name}</h1>
                )
            }
        3) 스프레드 연산자
            function Parent() {
            let info = {
                name : "홍길동",
                age: "20",
                address: "서울시"
            }
                return(
                    <Child {...info} />
                )
            }
            function Child({name, age}) {
                return(
                    <h1>Hello~ My name is {name}, {age}</h1>
                )
            }
        4) Props 로 컴포넌트 전달하기
            function PropsComp() {
                return (
                    <h1>PropsComponent</h1>
                )
            }
            function Parent() {
                return(
                    <Child>
                        <PropsComp />
                    </Child>
                )
            }
            function Child({ children }) {      // 소문자여야 함.
                return(
                    <h1>Hello~</h1>
                    {children}
                )
            }

8. 이벤트 처리하기
    (1) 이벤트란?
        - 웹 사이트에서 발생하는 사용자의 행위
            예) 버튼 클릭, 텍스트 입력, 페이지 스크롤, ... 행위가 발생 => '이벤트가 발생했다' 고 표현함.
        - 이벤트 핸들링 : 이벤트 발생 시 특정 코드가 처리하도록 만드는 작업 (처리하는 함수로 연결해주는 작업)
        - 이벤트 핸들러 : 이벤트를 처리하는 함수(객체)

    (2) HTML vs React 이벤트 핸들링 비교
        - HTML : onclick(소문자) -> onclick = "handleClick()"       // 호출하는 거라 () 붙음
        - React : onClick(카멜케이스 타입) -> onClick = {handleClick}   // 가상돔에서 만들어지기 때문에 실제 돔 호출이 아니라서 () 붙이면 에러뜸
                                         -> onClick = { () => {} }  // 직접 처리 시 콜백 함수 사용
        예) HTML
            <button type="button" onclick="handleClick()" />
        예) React
            <button type="button" onClick={handleClick} /> ,
            <button type="button" onClick={ () => {} } />

    (3) 이벤트 객체 활용 - 이벤트 발생 시 호출되는 객체
        - 이벤트객체.target -> 이벤트 객체 target 요소. .target 은 . 붙었으니 프로퍼티
        - 이벤트객체.target.name, 이벤트객체.target.innerText 등...
        예)
            const handleClick = (e) => {            // e : 여기에 뭘 쓰면 아래 onClick에서 handleClick 으로 보낼 때 받는다는 의미. 비워두면 안 받음.
                console.log(e.target);
            }
            
            <button type="button" name="버튼1" onClick={handleClick} />
            <button type="button" name="버튼2" onClick={ (e) => { console.log(e.target) } } />

    (4) 이벤트 객체를 활용하여 자식 -> 부모 에게 값을 전달 가능함 (나중에 진도 나간 부분)
        function 부모() {
            const handleParent(예시) = (4️⃣result(아무거나 변수명)) => { // 자식에게 받은 100을 result 로 정의
                5️⃣console.log(result);
            }
            return(
                <자식 1️⃣click={handleParent} />      // 이거로 자식에게 부모의 이벤트 주소가 넘어감.
            )
        }
        function 자식(2️⃣{click}) {
            const handleChild(예시) = () => {
                // 부모에게 100 전달
                3️⃣click(100);        // 부모에게 받은 click 에 100을 넣음
            }
            return(
                <button type='button' onClick={handleChild}>버튼</button>
            )
        }

9. 상태 관리(State) - useState
    (1) 컴포넌트가 관리하는 동적값이 변경되면, 리액트 컴포넌트가 자동으로 다시 렌더링함
    (2) useState 사용법
        [형식] import { useState } from 'react';
            const [(동적 값을 저장한)변수, set변수명] = useState(변수의 초기값);

        예) 숫자일 경우 : const [number, setNumber] = useState(0);
            문자일 경우 : const [string, setString] = useState('');
            배열일 경우 : const [array, setArray] = useState([]);
            객체일 경우 : const [obj, setObj] = useState({});

10. Ref - useRef
    (1) Ref란?
        - 실제 DOM 요소를 직접 참조하여 관리하는 기능을 가짐 (= Reference)
    (2) useRef 사용법
        import { useRef } from 'react';
        const nameRef = useRef(null);       // const 로 변수 정의. 객체기 때문에 초기 주소값은 null (undifined X)
        const handleOnChange = () => {}
        <input type="text" name="name" ref={nameRef} onChange={handleOnChange} />

            <비교> js / html 형식
            const name = document.querySelect('#name);
            name.addEventListener('', () => {});

    (3) DOM 입력폼 값 가져오기 : nameRef.current.value
        DOM 입력폼 포커스 정의 : nameRef.current.focus()

11. 리액트 라이프 사이클 & 리액트 개발자 도구(react developer tools)
    (1) 라이프 사이클 3단계 - 리액트에서 알아서 진행됨.
        - Mount(마운트) : 처음 생성
        - Update(업데이트) : 변화, 업데이트
        - UnMount(언마운트) : 종료, 제거
    (2) Mount(마운트)
        - 컴포넌트가 처음으로 렌더링되는 시점
        - 예_API 호출, 초기 데이터 렌더링 등
    (3) Update(업데이트)
        - 컴포넌트가 리렌더링 되는 시점
        - 예_State 변경, props 변경, 부모 컴포넌트 변경 등 ...
    (4) UnMount(언마운트)
        - 컴포넌트가 화면에서 사라질 때
        - 예_이벤트 종료 시점, 타이머 제거, 메모리 정리 등 ...
    (5) ** useEffect 로 라이프 사이클 제어
        형식 : 
            import { useEffect } from 'react';
            export default function [컴포넌트명] () {
                useEffect( () => {
                    // 1. 마운트 or 업데이트 시 출력
                    console.log("Mount or Update!!");

                    // 2. 언마운트 (형식일 뿐이지 평소 많이 안 쓰임)
                    return () => {
                        console.log("UnMount!!");
                    }
                } , [depencies, ...] );     // depencies 는 없으면 생략 가능.
                return(
                    <></>
                )
            }

12. SPA - Single Page Application
    (1) 개념 : 한 개의 HTML 페이지에서 모든 화면의 전환이 이루어짐 
        - 페이지 이동 시 새로고침 현상이 발생하지 않음
    (2) MPA(Multi Page Application) <-> SPA
        - 메뉴 별 독립적인 페이지를 생성하여 화면의 전환이 이루어짐
        - 페이지 이동 시 새로고침 현상이 발생함
    (3) 라우팅(Routing)
        - 메뉴로 구성된 컴포넌트 페이지로 이동하는 방식 // MPA 의 경우 페이지 마다 라우팅됨. SPA 의 경우 컴포넌트 마다 라우팅됨.
        - react-router-dom 패키지 : Link, NavLink 컴포넌트 포함
